Operator Overloading



Operators are overloaded by writing 
a function definition
(with a header and body)

as we normally would,except
the function name now becomes 
the keyword perator
followed by the Symbol for the operator 
being overloaded




for example,
the function name operator+ would be used to 
overload the addition operarator(+)


void operator ++()

The declration Syntax tells the compiler
to call this member function
whenever the ++ operator is encountered
provided the operand(the variable
operated on by the ++)
is of type Counter(class)

Overload operators to perform 
the same function
or similar functions on class objects
as the operators perform on objects 
of built-in types.

At last one argument of an 
operator function
must be a class object.

This prevents programmers 
from changing how
operators work on built-in types




Exception


The assignment operator(=) may be used
with every class without 
explicit overloading

The default behaviour of 
the assignment
operator is a memberwise 
assignment of
the data members of the class


The address operator (&) may also
be used with objects of any class without
overloading;it simply returns 
the address of
the object in memory.

The address operator can also be overloaded.




Overload operators using member functions.


when an operator function is 
implemented as a
member function,
the leftmost (or only) operand
must be a class object 
(or a reference to a class object)
of the operator's class.





#include <iostream.h>


class three_d {
  int x, y, z; // 3-D coordinates
public:
  three_d() { x = y = z = 0; }
  three_d(int i, int j, int k) {x = i; y = j; z = k; }

  three_d operator+(three_d op2); 
  three_d operator=(three_d op2); 

  void show() ;
};

// Overload +.
three_d three_d::operator+(three_d op2)
{
  three_d temp;

  temp.x = x + op2.x; // These are integer additions
  temp.y = y + op2.y; // and the + retains is original
  temp.z = z + op2.z; // meaning relative to them.
  return temp;
}

// Overload assignment.
three_d three_d::operator=(three_d op2)
{
  x = op2.x; // These are integer assignments
  y = op2.y; // and the = retains its original
  z = op2.z; // meaning relative to them.
  return *this;
}

// Show X, Y, Z coordinates.
void three_d::show()
{
  cout << x << ", ";
  cout << y << ", ";
  cout << z << "\n";
}

int main()
{
  three_d a(1, 2, 3), b(10, 10, 10), c;

  a.show(); //1, 2, 3
  b.show(); //10, 10, 10

  c = a + b; // add a and b together
  c.show();  //11, 12, 13

  c = a + b + c; // add a, b and c together
  c.show();  //22, 24, 26

  c = b = a; // demonstrate multiple assignment
  c.show();  //1, 2, 3
  b.show();  //1, 2, 3

  return 0;
}

// Use a friend to overload <<.
#include <iostream.h>

class three_d {
  int x, y, z; // 3-D coordinates - now private
public:
  three_d(int a, int b, int c) { x = a; y = b; z = c; }
  friend ostream &operator<<(ostream &stream, three_d obj);
} ;

// Display X, Y, Z coordinates - three_d inserter.
ostream &operator<<(ostream &stream, three_d obj)
{
  stream << obj.x << ", ";
  stream << obj.y << ", ";
  stream << obj.z << "\n";
  return stream;  // return the stream
}

int main()
{
  three_d a(1, 2, 3), b(3, 4, 5), c(5, 6, 7);

  cout << a << b << c;

  return 0;
}

// Overload [].
#include <iostream.h>


const int SIZE = 3;

class atype {

int a[SIZE];

public:
atype();

int operator[](int i)
{
return a[i];
}

};

atype::atype()
{

register int i;
for(i=0; i<SIZE; i++) a[i] = i;

}



int main()
{


atype ob;
cout << ob[2];  // displays 2
return 0;
