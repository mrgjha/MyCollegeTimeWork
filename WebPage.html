Introduction to Web Page

Web pages can be either static or dynamic A static Web page is an HTML document whose content is established at the time the page is created
A dynamic Web page is interactive; it can accept information from the user and also retrieve information for the user Whereas a static Web page is simply an HTML document, a dynamic Web page usually requires both an HTML document and a script

A script is a set of instructions written in a scripting language that tells a computer how to perform a task
Several technologies are available for creating dynamic Web pages; examples include CGI, Active Server Pages (ASP), and Java applets
CGI, which stands for Common Gateway Interface, is the protocol that allows a Web server to communicate with CGI scripts
  • mechanism for a Web browser to send data to a Web server
  • allow browser to submit data to a program running on the server
  • program is often called a ‘CGI script’
  • typically written in Perl, PHP or ASP
  • can also be a ‘real’ program (e.g. written in C)

A CGI script is a script that follows the standards specified by the CGI protocol The most widely used language for creating CGI scripts is Perl (Practical Extraction and Report Language)

How the World Wide Web works

Three parts that enable a user to browse a web page:
  • The web server that holds the information
  • the client that is viewing the information
  • the protocol that connects the two entities web server
Most popular browers (Windows):
  • Microsoft Internet Explorer
  • Netscape Navigator
  • Opera
  • Mozilla
Software Concept of Web servers:
  • a program that serves the files that form web pages (i.e. HTTP-, sound-,graphics- and multimedia files) to web users;
  • following the client / server model, the web server acts as the server and the web surfer’s web browser acts as the (remote) client, forwarding the surfer’s requests to the web server;
  • every web site to be published on the internet must be hosted by an online web server;
  • Web servers are often bundled with a larger package of internet- and intranetrelated application programs for serving e-mail, downloading requests for File Transfer Protocol (FTP) files etc., and building and publishing web pages.

The HTTP Response
The Response Line requires three items of information:

  • The HTTP version number
  • An HTTP request code that reports the success or failure of the request
  • A short message describing the status of the request

Cookies

  • remember preferences using cookies
  • Cookies are useful for storing user info that should be retained from one page to the next.
  • Cookies are written to the client’s hard drive.
  • developed by Netscape but widely adopted
  • typically, cookies are only shared with the server that set them
  • support for cookies in Perl, PHP and ASP
